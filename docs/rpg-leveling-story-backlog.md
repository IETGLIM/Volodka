# RPG: уровень персонажа и опыт — бэклог, привязка к истории

Документ дополняет общий план обхода (`docs/exploration-expert-incremental-backlog.md`). Статусы: **сделано** | **в работе** | **запланировано**.

## Нарративная рамка

Опыт — это **наблюдательность и прожитый сюжет**: новые узлы истории, собранные стихи, завершённые квесты. Уровень — **зрелость героя** в мире Володьки; очки навыков за уровень остаются основным «механическим» вознаграждением за рост.

## Слой R0 — ядро и интеграция (сделано в текущей итерации)

| ID    | Задача | История / смысл | Файлы / заметки |
| ----- | ------ | ---------------- | --------------- |
| R0.1  | Кривая XP и потолок уровня | Ограничение прогрессии сюжетом, а не бесконечным гриндом | `src/lib/rpgLeveling.ts`, `rpgLeveling.test.ts` |
| R0.2  | Поля стейта: уровень, XP, до следующего уровня | Сейвы и HUD отражают «рост героя» | `src/data/types.ts`, `src/shared/types/game.ts`, нормализация в `gameStore` |
| R0.3  | Начисление XP: первый визит узла сюжета | Исследование карты = опыт наблюдения | `setCurrentNode` → `addExperience(6, story_node:…)` |
| R0.4  | Начисление XP: первый сбор стиха | Артефакты творчества усиливают героя | `collectPoem` → `addExperience(22, poem:…)` |
| R0.5  | Награды квестов: поле `experience` + дефолт при завершении | Завершённые дуги сюжета дают заметный скачок | `QuestReward`, `quests.ts`, `completeQuest` в `gameStore` |
| R0.6  | Событие `player:level_up`, тост при повышении уровня | Игрок явно чувствует веху в истории | `EventBus.ts`, `addExperience` в `gameStore` |
| R0.7  | HUD: уровень и полоса XP | Читаемость прогресса без отдельного экрана | `HUD.tsx` (`CyberLevelXpBar`) |

## Слой R1 — баланс и контент (запланировано)

| ID    | Задача | История / смысл |
| ----- | ------ | ---------------- |
| R1.1  | Проход по квестам и узлам: таблица XP vs целевой темп главы | Чтобы финал главы ощущался на «правильном» уровне без обязательного фарма |
