# Volodka — технический директор: дорожная карта AAA-качества (Web / Vercel)

Цель: структурировать работу по **реалистичному масштабу**, **интерьеру в духе Blade Runner**, **интерактиву**, **квестам** и **ассетам** без иллюзии «одного PR». Статусы этапов: **P0** (фундамент) → **P4** (контент и полировка).

**Платформа:** Next.js + R3F + Rapier на Vercel. **WebGL** — основной путь сейчас; **WebGPU** (Three.js `WebGPURenderer` / будущие пайплайны) — отдельный **спайк** после стабилизации P0–P1 (браузерная матрица, размер бандла, fallback).

Связанные документы: `docs/exploration-expert-incremental-backlog.md`, `docs/rpg-leveling-story-backlog.md`.

**Цель:** единая система «метры сцены = метры дизайна», без колоссов и щелей.

| ID    | Задача | Файлы / зона | Критерий готовности |
| ----- | ------ | ------------- | ------------------- |

## Риски и ограничения Vercel

- Длинные сессии WebGPU и тяжёлые compute-shader пайплайны — проверять **Chrome/Edge/Safari**, лимиты serverless не относятся к клиенту, но **размер чанка** критичен.
- Всё, что «полноценный Blender-интерьер» в одном PR, противоречит стабильности; только инкременты с чеклистом регрессии (см. `E5.1` в exploration backlog).
